# Contributing

Thank you for improving Easy DevOps.

1. Open an issue describing the problem or proposed change.
2. Create a focused branch from `main`.
3. Keep scripts compatible with the supported systems in the README.
4. Run the quality workflow locally where possible.
5. Use English Conventional Commit subjects.
6. Open a pull request with test evidence and documentation updates.

Never include credentials, private keys, generated environment files, ACME data,
logs, or backups. Security reports must follow [SECURITY.md](SECURITY.md).
